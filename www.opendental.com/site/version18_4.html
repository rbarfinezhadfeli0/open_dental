Version 18.4

See Versions

Version 18.4 was released on 03/20/2019. EHR certified 2014 edition.

Overview of Version 18.4.

Changes by Category

Appointments Module
  • Automatically detach treatment planned procedures from completed appointments. Edit Appointment
  • When block appointment scheduling blockout types are added on top of an existing standard blockout type, the standard blockout type will be removed. Appointment Module Preferences, Blockouts
  • Enhancements to Unscheduled List. Unscheduled List, Appointments Module Preferences
  • Filter ASAP list by procedure code. ASAP List
  • Add functionality to the Recently Contacted tab in Recall. Recall List
  • Prompt user when making an appointment if an unscheduled appointment exists. Unscheduled List
Chart Module
  • Procedure Status for In Process. Chart Module, Procedure Buttons, QuickTip: In Process Status for Multi-Visit Treatment
  • Easily chart quadrant procedures. Enter Treatment
  • Right click procedure to set to EO or EC status. Chart Module
  • Additional fields added to the Edit All Procs window. Chart Module
eServices
  • Edit and send eConfirmations, eReminders, and Patient Portal invites as HTML. Automated Messaging Feature
  • Filter the mobile web pharmacy list by clinic. Mobile Web Pharmacies
  • Text Messaging window can now group by phone number. Text Messaging Mailbox
  • Mobile friendly Web Forms. Web Forms: What Patient Sees
  • Mobile friendly Patient Portal. Patient Portal: What Patient Sees
Insurance
  • ERA write-offs posted by plan type. Manage Module Preferences, ERA
  • Split Claim option added to ERA payment window. ERA
  • View ERA'a and EOB's from Edit Claim window. Claim
  • Enhancements to frequency limitations. Insurance History
Preferences
  • Additional options for themes. Miscellaneous Setup
  • Preference to not recalculate insurance estimates when a claimproc is Received. Account Module Preferences
  • Preference for number of days to automatically refresh the Appointment Module. Account Module Preferences
  • Preference to select payment type for recurring charges. Account Module Preferences, CC Recurring Charges
  • Preference to remove All as an option for many clinic boxes. Enterprise Setup
  • Preference to bill UCR fee and mark non-covered services as "do not bill to insurance". Family Module Preferences, Other Benefits
  • Setup window for enterprise users. Enterprise Setup
  • Automate Repeating Charges tool. Repeating Charge, Account Module Preferences
Security
  • Permission for Insurance Verification. Permissions, Insurance Verification List
  • Permission for Production and Income - View All Providers now applies to graphical reports. Permissions, Graphic Reports
  • Improve security in the Log On window. Log On
  • Enhanced audit trail entries for blockouts. Permissions, Blockouts
  • Prompt to create a username and password for MySQL. MySQL Security
Sheets
  • Enhancements to default Medical History sheet. Medical History Layout
  • Enhancements to Exam Sheets. Exam Sheet
Miscellaneous
  • Wireless phone and text options added to Add Family window. Add Family
  • Export accounting entries and reconciles. Accounting, Reconcile
  • Computer name removed from Email Inbox window. Email Setup, Email Client
  • Transworld Systems, Inc. message history window added. TSI History
  • Connect to older versions of Open Dental databases from a workstation without upgrading the database to the current workstation version. Choose Database
  • Archive tool uses verification process to prevent archiving to the wrong database. Backup Tool
  • Tool to mark unused Fee Schedules as hidden. Fee Schedules
  • Integration with DentalXChange's Patient Credit Score Services. DentalXChange Patient Credit Score Service
  • Credit Card Edit window allows quick paste notes. Right Click Text Box
  • FHIR users can update appointment confirmation statuses. API Setup
  • Automation action to change patient status.Automation
  • Spell check improved. Spell Check
  • Prevent blank Rx sheets from printing. Rx Manage
  • Message appears to warn a user when allocating more unearned income than is available on the account. Allocate Unearned Income
