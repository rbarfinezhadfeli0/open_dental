Version 18.2

See Versions

Version 18.2 was released on 09/21/2018. EHR certified 2014 edition.

.NET 4.5.2 is required. See .NET to upgrade (Windows 7 users).

Major Changes

  • Enter a term date for a provider to restrict sending claims or prescriptions, and scheduling or completing appointments. Provider
  • New Patient Status Setter tool to inactivate patients that have not been in for a specified time. Patient Status Setter
  • ACH option added for Pay Simple. PaySimple Add Card
  • Income transfer manager. Income Transfer
  • 2018 ICD 10 codes. ICD-10 Codes
  • Integration with DentalXChange Attachment Service. DentalXChange Attachment

Changes by Category

Account Module

  • ACH option added for Pay Simple. PaySimple Add Card
  • Closing a payment plan will no longer remove remaining future adjustments. Close Payment Plans
  • Provider column added to Payment Plan Procedures and Credits window. Payment Plan Procedures and Credits
  • Warning occurs when a procedure's provider is different from the payment plan's provider. Old Payment Plan
  • Speed enhancements. Account Module
  • Preselect specific payment plan charges or adjustments when entering a new payment. Payment
  • XCharge, PayConnect, and PaySimple can be enabled at the same time. Credit Card Payment
  • Prevent saving new cards option determines whether new card information can be saved and whether new transactions can be processed. XCharge Setup, PayConnect Setup, PaySimple Setup
  • Credit card security code (CVV) and zip code can be entered on PayConnect transactions saved tokens. PayConnect Window
  • Option to automate recurring charges. https://www.opendental.com/manual182/modulesetupaccount.html
  • Recurring charges will now stop when Open Dental is shut down. CC Recurring Charges

Appointments

  • Add NetProduction to an appointment view to show Daily Production in the Appointments module. Appointment View Edit
  • Speed enhancements for clinic appointment search.
  • Search results limited to providers on the schedule for the selected appointment view. Pinboard
  • Add broken appointments to the ASAP list. Break Appointment, ASAP List
  • Set a default date range in the unscheduled list. Unscheduled List

Clinics

  • Unrestricted Search permission allows users who are restricted to a specific clinic to search all patients in the database. They can also change a patient's clinic from the Edit Patient Information grid. Permissions
  • Time required to generate statements for multiple clinics has been reduced. Billing List
  • Clinic filter when retrieving web forms. Web Forms Retrieve
  • Limit to Ops in Clinic option added to Schedule Setup window. Quickly selects all providers and employees associated with the selected clinic. Schedule Setup
  • Options to Clear All Blockouts for Day, Op only, Limit to Ops in Clinic Filter, or Clear All Blockouts for Day, Clinic only added to right-click menu on Appointments. Blockouts

eServices

  • Patients cannot be sent more than one eConfirmation text in the same day. eConfirmations
  • When logging on to the Mobile Web, Clinic defaults to the clinic last active when the user logged off or closed Open Dental. Mobile Web Home
  • Option to enable/disable electronic signatures on all web forms. Web Form Setup
  • Option to enable/disable typed signatures on certain web forms by modifying the URL. Web Form Setup
  • Option to exclude a patient from eConfirmations and eReminders. Edit Patient Information
  • Time slot finding logic enhanced for the Web Sched appointment search.
  • Web Sched Recalls can now be sent out for custom recall types. Web Sched Recall
  • Include an opt-out statement in eService emails that includes practice/clinic address and instructions to unsubscribe. Email Setup
  • When using a StatementURL, patient will be directed to the Patient Portal sign-in window. Payment Portal
  • Appointment time patterns are preserved for appointments created from the mobile web feature. Mobile Web Operatories
  • Interface changes to the Patient Portal setup tab. Patient Portal
  • Option to add a clinic ID to the patient portal URL. Patient Portal
