<p>For <a href="canada.html">Canada</a> users, see the information below to assist with setup and installation.</p>
<h2>General</h2>
<p>To utilize all Canada-specific features, the <a href="regionlanguagesettings.html">Region and Language</a> settings on all workstations must be set to <i>Canada</i>.</p>
<h2>Claims</h2>
<p>The Canadian Dental Association (CDA) claim form is set as the default. </p>
<p>Claims needed for an American patient must be sent via a paper claim using the American Dental Association (ADA) claim form. We are not allowed to distribute the ADA claim form by law, so it must be purchased from the ADA instead. </p>
<p>Dentists who send electronic claims use <a href="itrans20.html">ITRANS 2.0</a>. Hygiene-Only offices who send electronic claims use <a href="claimstream.html">Claimstream</a> (also known as the TELUS integration). Hygiene providers do not need to sign up with Instream.</p>
<h2>Tooth Numbering/Surfaces</h2>
<p> In <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7preferences.html">Preferences</a>, the <i>Tooth Nomenclature</i> preference is set to FDI Notation. Supernumerary teeth can be entered as 99.</p>
<p class="MarginBottomZero">In the United States: </p>
<ul class="MarginBottomGap">
<li>Valid tooth surfaces are MOIDBLF (facial). See <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7entertreatment.html">Enter Treatment</a>.</li>
<li>We use V to represent a class 5 location along the gingival margin of the B or F.</li>
</ul>
<p class="MarginBottomZero">In Canada: </p>
<ul class="MarginBottomGap">
<li>Valid tooth surfaces are MOIDBLV (vestibular).</li>
<li>Class 5 is represented by 5 instead of V to avoid confusion with vestibular.</li>
</ul>
<h2>Clinics</h2>
<p>If a practice is using the <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7clinics.html">Clinics</a> feature, make sure to use a different provider for each dentist/clinic combination. Each dentist is assigned an office number on the Edit Provider window. See, <a href="canadaprovider.html">Canada Provider Setup</a>. So if a dentist is at two offices, the only way to enter two office numbers is to use two separate providers.</p>
<h2>Privacy Laws</h2>
<p>Open Dental only stores personal data for a short time when running conversions or for web forms not yet collected by the office. This information is encrypted, secure, and deleted after the amount of time required to perform the service.</p>
