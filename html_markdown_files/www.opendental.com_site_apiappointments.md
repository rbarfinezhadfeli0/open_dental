<a name='Appointments GET Slots'></a><h2>Appointments GET Slots</h2>
<p>Version Added: 21.1</p>
<p>For simple dental office setups. See also <a href="apischedules.html">Schedules GET</a>.</p>
<p>This is closer to how search behaves from within Open Dental instead of WebSched. This requires no advanced setup. It looks at open schedule times, whether the schedules are attached to a provider or an operatory.</p>
<p>There are, however, some differences between this and the Search in Open Dental. This returns entire open slots instead of a series of suggested appointment times or the first appointment time for each day. It also currently searches by entire appointment length instead of the XXX provider times on appointments. For providers assigned to a single operatory, the results will be the same as what a person would see if looking at the Appointments Module. If a provider has existing appointments in multiple operatories, it considers all of them as a whole and only returns slots that are available in all operatories for that provider simultaneously. </p>
<p>Most users will specify a <b>ProvNum</b> and an <b>OpNum</b>.</p>
<p><b>Parameters:</b> All optional.<br></p>
<p><b>date:</b> Search for a single day in "yyyy-MM-dd" format. Must be today or a future date. Default is the next 14 days.<br><b>dateStart, dateEnd:</b> Search for a date range, inclusive of both dates, in "yyyy-MM-dd" format.<br><b>lengthMinutes:</b> Ignores slots that are shorter than the supplied number of minutes.<br><b>ProvNum:</b> ProvNum of a provider. Defaults to the Practice Default Provider.<br><b>OpNum:</b> (Added in version 22.1) FK to operatory.OperatoryNum that the provider is assigned to.<br></p>
<p><b>Example Requests:</b><br> GET /appointments/Slots?date=2021-02-15<br> GET /appointments/Slots?date=2022-03-30&amp;ProvNum=50&amp;OpNum=16<br> GET /appointments/Slots?ProvNum=1&amp;OpNum=18&amp;dateStart=2022-03-28&amp;dateEnd=2022-04-01<br></p>

<a name='Appointments GET SlotsWebSched'></a><h2>Appointments GET SlotsWebSched</h2>
<p>Version Added: 21.1</p>
<p><span style="color:red">Rarely used. Probably just use Appointments GET Slots instead. Offices must be subscribed to WebSched eServices.</span></p>
<p>This gets slots according to the existing logic used for WebSched existing patient or new patient. Customer must first have gone to Setup WebSched (Existing Patient or New Patient) and done all the setup, including creating Appointment Types and linking Appointment Types to WebSched. This all requires a fair amount of setup on the customer's end, but that restriction is usually preferred when patients are making their own appointments.</p>
<p>After calling this method, use Appointments POST WebSched to create a new WebSched appointment.</p>
<p><b>date:</b> Search for a single day in "yyyy-MM-dd" format.<br><b>dateStart, dateEnd:</b> Search for a date range, inclusive of both dates, in "yyyy-MM-dd" format. If no dates at all are supplied, the default is the next two weeks.<br><b>ClinicNum:</b> Required if clinics are enabled. FK to clinic.ClinicNum.<br><b>defNumApptType:</b> Optional. definition.DefNum where definition.Category=42 (NewPat) or 51 (ExistingPat). There must also be an appointmenttype linked to that DefNum and it must be set to be used in WebSched.<br><b>isNewPatient: </b>Optional. Defaults to false. Should match the kind of <b>defNumApptType</b> that you use.<br></p>
<p><b>Example Request:</b><br> GET /appointments/SlotsWebSched?dateStart=2021-02-15&amp;dateEnd=2021-02-15&amp;defNumApptType=326<br></p>
<p><b>Example Response:</b><br><span class="codeblock"> [<br>  {<br> "DateTimeStart": "2021-02-15 08:00:00",<br> "DateTimeEnd": "2021-02-15 08:30:00",<br> "ProvNum": 1,<br> "OpNum": 1<br> },<br>  {<br> "DateTimeStart": "2021-02-15 08:30:00",<br> "DateTimeEnd": "2021-02-15 09:00:00",<br> "ProvNum": 1,<br> "OpNum": 1<br> },<br>  {<br> "DateTimeStart": "2021-02-15 09:00:00",<br> "DateTimeEnd": "2021-02-15 09:30:00",<br> "ProvNum": 1,<br> "OpNum": 1<br> },<br> etc...<br> ]<br></span></p>

<a name='Appointments POST'></a><h2>Appointments POST (create)</h2>
<p>Version Added: 21.1</p>
<p>Creates an appointment for a patient. </p>
<p><b>PatNum: </b>Required.<br><b>Op:</b> Required.<br><b>AptDateTime: </b>Required. String in "yyyy-MM-dd HH:mm:ss" format. Use Appointments GET Slots to find available times.<br></p>
<p><b>AptStatus:</b> Optional. Either "Scheduled", "Complete", "UnschedList", or (rarely used) "PtNote", "PtNoteCompleted". Default "Scheduled".<br><b>Pattern:</b> Optional. Normally, if more than 24 hrs notice is given, you would not specify <b>breakType</b>.
