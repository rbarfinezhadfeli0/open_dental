<p class="MarginBottomZero">Before using a Windows Tablet as a Kiosk device, consider the following issues. </p>
<ul class="MarginBottomGap">
<li>May run slower due to limited graphic capabilities</li>
<li>Screen size and configuration. On Window 8 Surface Pro tablets, the kiosk doesn't take up the whole screen. The writing can appear very small to patients and the orientation can also be shifted by movement of the tablet.</li>
<li>Tablets need to have their keyboard undocked when filling forms. Docked keyboards may resize the full screen windows and can cause issues with scroll bars.</li>
<li>Tablets need to be in tablet mode for the touch keyboard to automatically popup when using the kiosk.</li>
<li>Tablets are very portable and can easily be lost or misplaced in a reception area setting.</li>
</ul>

<h2>Limitations</h2>
<p>Android is not a 'controlled ecosystem'. With Android, you can get new devices with old software that will not work well. </p>
<p><b>The following are not supported</b>:  <li>Fire tablets, which run Fire OS, are not compatible with eClipboard or ODTouch.</li>
<li>Android 4.4 KitKat</li>
<li>Vasoun brand Android tablets are not compatible with eClipboard or ODTouch.</li>
</p>
<h2>Remote Software &amp; Security</h2>
<p>There are software products available for remote connection to one of your office computers over an iPad or Android. The <a href="remote.html">Remote Access</a> page covers some of these products, but there are others.</p>
<p>See <a href="wireless.html">Wireless</a> for ideas regarding security when using a tablet device.</p>
<h2>Formerly Supported</h2>
<p>eClipboard was formerly supported for Windows tablet devices. These devices are no longer supported as of September, 2023.</p>
