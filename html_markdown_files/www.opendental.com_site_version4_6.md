<p>See <a href="versions.html">Versions</a></p>
<p>Beta version released 12/29/06 </p>
<p><b>E-claims:</b> Table added to permanently record and track each electronic claim transaction (needed for Canada, but useful for everyone)</p>
<p><b>Canadian E-Claims:</b> Significant progress has been made. Not done yet. See <a href="canada.html">Canada</a>. </p>
<p><b>Fee Schedules:</b> Export function. Import function. Global update fees for all patients. Search and fix insurance plans with wrong fee schedules. Allow decreasing fees by a percentage. Compare three fee schedules while editing. </p>
<p><b>General Accounting:</b> Automation of Retained Earnings, General Ledger Detail report, Acccounting Balance Sheet report.</p>
<p><b>Oracle:</b> Built-in support for Oracle. (should be considered beta)</p>
<p><b>Update enhancements:</b> The claim form and procedure codes can now be updated independently of the program itself. </p>
<p><b>ADA 2006 ClaimForm:</b> This version is capable of importing and using the ADA 2006 claimform, although without a background. That download is freely available starting with 4.6.29. See <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7update.html">Update</a>. In order to include the background, we need to wait for a licensing agreement with the ADA. That download will cost approximately $22. As a stopgap measure, we can mail blank claimforms to offices as needed. </p>
<p><b>NPI: </b> Included and functional with e-claims clearinghouses.</p>
<p><b>1500 Claimform:</b> New medical claimform is supported, and a free update is available. It will be posted very soon. </p>
<p>Patient language field added. <br> It now asks before automatically applying part of a payment to other family members. <br>  Multiple questionnaires allowed for each patient. Accessed from commlog. <br>  Appointment search feature now includes searching by time also.<br>  Right click to delete blockout.<br>  Check numbers always show in Account module.<br>  Right click to delete rows in Progress Notes.<br>  Onscreen keyboard added in the select patient window for touchscreen users.<br>  Select Patient window: Only one phone search field which looks at all phone numbers and ignores non-numbers. <br>  Select Patient window hides deceased and archived patients by default.<br>  Contact preferences tracked separately for recall, regular, and confirm. Integrated into various screens.<br>  Allowed to edit patient info within recall detail window.<br>  Confirmation list remembers last patient selected.<br>  Can print recall list and confirmation list as displayed on screen.</p>
