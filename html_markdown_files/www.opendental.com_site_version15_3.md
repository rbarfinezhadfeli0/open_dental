<p>See <a href="versions.html">Versions</a>.</p>
<p>Version 15.3 was released on 11/18/2015. <br></p>
<p class="MarginBottomZero"><b>Integrated Texting:</b> A two-way text messaging service hosted by Open Dental. This is a paid service. <a href="textintegrated.html">Integrated Texting</a></p>
<ul class="MarginBottomGap">
<li>Receive free and unlimited text messages from patient to office.</li>
<li>Send text messages for a per message fee. </li>
<li>Dedicated phone numbers for each clinic or office using Open Dental.</li>
<li>Patients can respond with no time limit. </li>
<li>CommLogs automatically created for all messages. </li>
<li>Set monthly text message limits by location. </li>
<li>Send confirmation text messages directly from the <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7appointments.html">Appointment Module</a>. </li>
<li>View text log for selected messages. Reply to selected message from text log.</li>
</ul>
<p class="MarginBottomZero"><b>Fee Schedules</b>:  </p>
<ul class="MarginBottomGap">
<li>New fee schedule preference determines whether fees apply globally to all providers/clinics or can be changed for specific providers/clinics. <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7feeschedules.html">Fee Schedules</a></li>
<li>Associate procedure fees in a fee schedule to specific providers and/or clinics. <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7procedurecodes.html">Procedure Code List</a></li>
<li>New definition for fee colors. <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7definitionsfeecolors.html">Definitions Fee Colors</a></li>
</ul>
<p class="MarginBottomZero"><b>Web Sched</b><b>Enhancements:</b><a href="websched.html">Web Sched</a></p>
<ul class="MarginBottomGap">
<li>Settings that determine available appointment slots can be previewed and edited on the eServices Setup, Web Sched tab. <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7webschedrecallsetup.html">Web Sched Recall Setup</a></li>
<li>Only operatories flagged as 'Is Web Sched' are considered when searching for available time slots. <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7operatories.html">Operatory Setup</a></li>
<li>Set provider considerations when determining available time slots (first available, primary, secondary, last seen hygienist).</li>
<li>Available time slots are based on appointment length (no longer hour blocks).</li>
<li>Start/end times are based on time increments of the schedule (no longer limited to on-the-hour start times). </li>
</ul>
<p class="MarginBottomZero"><b>Central Enterprise Management Tool</b>: <a href="central.html">Central Enterprise Management Tool</a></p>
<ul class="MarginBottomGap">
<li>Search for patients from the main CEMT window. <a href="central.html">Central Enterprise Management Tool ( CEMT )</a></li>
<li>Log in to an Open Dental database from CEMT. <a href="central.html">Central Enterprise Management Tool</a></li>
<li>Sync all security information for multiple databases simultaneously. <a href="cemtsecurity.html">CEMT Security</a></li>
<li>Run Production and Income report by provider across multiple databases. <a href="cemtprodincreport.html">CEMT Production and Income Report</a></li>
<li>Multi-threaded connection speeds up the retrieval of data from multiple databases. </li>
</ul>
<p><b>Payment Plan</b>: Recalculate amortization schedules for prepayments or payments to principal while preserving historical payments. <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7paymentplanpatient.html">Old Payment Plan</a></p>
<p class="MarginBottomZero"><b>Non-Dental Practices or Clinics</b>: Set a practice or clinic as "non-dental". <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7clinicsetup.html">Clinic Setup</a><a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7practice.html">Practice Setup</a><br>  If set as non-dental, most tooth and surface references are removed from the interface. <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7nondentalinterface.html">Non-Dental Practice/Clinic</a></p>
<ul class="MarginBottomGap">
<li>Graphical tooth chart removed from the module</li>
<li>Tabs for missing teeth, movements and primary removed from the module</li>
<li>Buttons for Perio Chart, Tooth Chart and Ortho Chart removed from the Chart Module toolbar. </li>
<li>Labels that reference teeth or surfaces are hidden throughout the software</li>
<li>Procedure Code column widened in Reports. </li>
</ul>
<p><b>ICD-10 Codes</b>: New preference to use ICD-10 instead of ICD-9 diagnosis codes on procedures. <a href="../autoLogin.aspx%EF%B9%96ReturnUrl=%EA%A4%B7help253%EA%A4%B7icdcodes.html">ICD Diagnosis Codes</a></p>
