Definitions: Image Categories

In the [Main Menu](mainmenu.html), click Setup, [Definitions](definitions.html), Image Categories.

![](images/definitionsImageCats.png)

The Image Categories definition determines the category folders available to organize documents in the [Imaging Module](images.html).

Also with this definition, set category images to show in other areas of the program and set the default category where documents are saved (e.g., saved treatment plans, statements, etc).

Double-click an existing item or click **Add** to create a new item. The Edit Definition window will open.

![](images/definitionEditImageCats.png)

* **Name**: Enter the category name.
* **Usage**: (Optional) Set the category properties by checking the boxes that apply. Letters in parentheses display the designation used in User Queries.
  + **Show in Chart Module**: Check to show category as both a folder in the Imaging Module and as an additional tab at the bottom of the Chart Module.
  + **Show Thumbnails**: Images in this category display a thumbnail in the Image Categories list. Only image files types (e.g., .jpg, .gif, .png, etc.) will show a thumbnail preview.
  + **Show in Patient Forms**: Check to access documents in this category from the [Patient Forms](patientform.html). Double-clicking a document in the forms window will immediately switch to the document in the Imaging Module.
  + **Show in Patient Portal**: Check to allow patients access to the documents in this category from the [Patient Portal Feature](../site/patientportal.html).
  + **Patient Pictures (only one)**: Check to use images stored in this category as the patient picture in the [Family Module](family.html) and on the [Appointments Module](appointments.html).
  + **Statements (only one)**: Check to store PDF copies of all printed and emailed [Statements](statement.html) in this category. Also stores statements exported as a CSV.
  + **Graphical Tooth Charts (only one)**: Check to store tooth charts saved from the Chart Module in this category.
  + **Treatment Plans**: Check to store PDF copies of printed, emailed, or signed treatment plans from the [Treatment Plan Module](treatmentplan.html) in this category. Enable the  *Save signed Treatment Plans to PDF* preference to automatically save signed treatment plans.
  + **Payment Plans (only one)**: Check to store PDF copies of printed payment plans or payment plans signed on eClipboard. See [Sign and Print Payment Plan](paymentplanprint.html) in this category.
  + **Claim Attachments (only one)**: Check to store attachments from [DentalXChange Attachment Service.](dentalxchangeattachmentservice.html)
  + **Lab Cases (only one)**: Check to store PDF copies of [Lab Slips](labslips.html) in this category when printed, emailed, or saved as a PDF.
  + **Autosave Forms (only one)**: Check to enable *Check 'Autosave' By Default* in [Sheet Def Properties](sheetproperties.html) and *Autosave to Images* in [Fill Sheet](sheetsfillout.html). Sheets are only autosaved when clicking OK from the Fill Sheet window and the *Autosave to Images* box is checked. Autosaved sheets or forms will be saved as a PDF to this category.
  + **Task Attachments (only one)**: Check to enable attachments on tasks. See [Task Attachments](taskattachments.html).
  + **Claim Responses (only one)**: Check to enable saving claim responses to the Imaging Module. Only used for Canada users.

Click **OK** to save.

To combine duplicate images, see [Merge Image Categories](mergeimagecategories.html).
