"Security Admin" Permission

The *Security Admin*Â [Permission](permissions.html) allows access to administrator functions. At least one user must have this permission. It allows/blocks a user from the following:

* Locking [Accounting](accounting.html) entries.
* Clean Up Allowed tool. [Fee Schedules](feeschedules.html)
* Access to [Procedure Code Tools](procedurecodetools.html).
* Access to [eCW Program Link](ecwbridgesetup.html).
* Access to create users and move patients in the [Providers](providers.html).
* Use Undo in the [Recall List](recalllist.html).
* Delete [Recall Types](recalltypes.html).
* Access to [Security](security.html).
* Access to the [Audit Trail](audittrail.html).
* Access to [Dental School Setup](dsusergroupdefaults.html) (Dental Schools).
* Access to the [Employees](employees.html) list.
* Access to the Etrans tool in [Database Maintenance](databasemaintenance.html).
* Make changes to Setup in [Update](update.html).
* Make changes to [eServices Signup](eservicessignup.html).
* Add IP Addresses in [Cloud Management](cloudmanagement.html) (Open Dental Cloud users only).
* Access to the [Patient Status Setter](patientstatussetter.html) tool.
* Using the Zero Claims button in the [Outstanding Insurance Claims Report](reportoutins.html).
* Access to certain [Database Maintenance Tools](databasemaintenancetools.html).

A SecurityAdmin log entry is also added to the Audit Trail when:

* A user clicks *Stop Monitoring* on the eServices Setup, [eConnector](econnector.html) Service window.
* A user edits permissions for a user group.
* The user group is changed for a user.
* The Registration Key is changed.
