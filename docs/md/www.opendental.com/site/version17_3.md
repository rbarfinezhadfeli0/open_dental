Version 17.3

See [Versions](versions.html).

Version 17.3 was released on 12/28/2017. EHR certified 2014 edition.

Major Features
 ASAP Appointments

* Send text messages about available openings from the ASAP List. [ASAP List](../manual/asaplist.html)
* Web Sched ASAP eService: Notify patients about last minute appointment openings to patients on the ASAP list and include a clickable link they can use to reschedule their appointment online. [Web Sched ASAP](websched.html)
  + [Send Web Sched ASAP Messages](../manual/webschedasap.html)
  + [What the Patient Sees](../manual/webschedasappatientsees.html)
* Appointment with a status of scheduled can also be marked ASAP. [Edit Appointment Window](../manual/aptedit.html)
* Right click from the appointment schedule to mark an appointment as ASAP. [Appointments Module](../manual/appointments.html)

Patient Payments [Patient Payments](https://www.opendental.com/manual/paymentpatient.html)

* Payment window now shows outstanding charges instead of family balances. [Payment window](../manual/payment.html)
* Pay split manager is incorporated into the Payment window, making allocation to procedures easier. [Patient Payment Allocation Options](https://www.opendental.com/manual/paymentpatient.html)
* Payment preferences determine behavior and requirements when entering a payment, such as allocation requirements, auto pay split suggestions, and pay split logic. [Patient Payment Preferences](https://www.opendental.com/manual/paymentpatientprefs.html)
* User is prompted for payment amount before entering Payment window. [Credit Card Payments](../manual/paymentpatientcreditcard.html), [Cash, Check, EFT, Other Payments](https://www.opendental.com/manual/paymentpatientother.html)
* Payment window displays the total value of selected outstanding charges. [Payment window](../manual/payment.html)
* Account module preference determines default logic for auto pay split suggestions: adjustments then FIFO, or FIFO only. [Account Module Preference](https://www.opendental.com/manual/modulesetupaccount.html)
* Account module preference for allowing future dated patient payments. [Account Module Preferences](https://www.opendental.com/manual/modulesetupaccount.html)
* Easily allocate unearned / unallocated income to procedures using the Payment dropdown, Allocate Unearned,. [Allocate Unearned Income to Procedures](../manual/unearnedallocate.html)
* Transfer income using the Payment dropdown, Income Transfer. [Income Transfers](https://www.opendental.com/manual/incometransfers.html)

Patient Portal

* Enhanced interface [What the Patient Sees](../manual/portalpatientsees.html)
* View scheduled and completed appointments, as well as action needed items such as Web Sched Recall notifications for appointments not yet scheduled and eConfirmation links for scheduled appointments not yet confirmed. [What the Patient Sees](../manual/portalpatientsees.html#appts)
* View saved treatment plans. [What the Patient Sees](../manual/portalpatientsees.html#tp)

User Queries: [User Queries](../manual/queryoverview.html)

* Mark favorite user queries as 'released' vs 'not released'. Released queries can be run by users with less security permissions. [Query Favorites](../manual/queryfavorites.html)
* When running a query favorite, user is prompted to set user variables. [Query Favorites](../manual/queryfavorites.html)
* User Query Admin permission allows/blocks user from writing or editing query favorites, running unreleased queries, adding or removing favorites, and marking favorite queries as released/not released. [Permissions](../manual/permissions.html)
* User Query permission changed. Allows/blocks user from running released queries. [Permissions](../manual/permissions.html)
