"Security Admin" Permission

The Security Admin Permission allows access to administrator functions. At least one user must have this permission. It allows/blocks a user from the following:

  • Locking Accounting entries.
  • Clean Up Allowed tool. Fee Schedules
  • Access to Procedure Code Tools.
  • Access to eCW Program Link.
  • Access to create users and move patients in the Providers.
  • Use Undo in the Recall List.
  • Delete Recall Types.
  • Access to Security.
  • Access to the Audit Trail.
  • Access to Dental School Setup (Dental Schools).
  • Access to the Employees list.
  • Access to the Etrans tool in Database Maintenance.
  • Make changes to Setup in Update.
  • Make changes to eServices Signup.
  • Add IP Addresses in Cloud Management (Open Dental Cloud users only).
  • Access to the Patient Status Setter tool.
  • Using the Zero Claims button in the Outstanding Insurance Claims Report.
  • Access to certain Database Maintenance Tools.

A SecurityAdmin log entry is also added to the Audit Trail when:

  • A user clicks Stop Monitoring on the eServices Setup, eConnector Service window.
  • A user edits permissions for a user group.
  • The user group is changed for a user.
  • The Registration Key is changed.
